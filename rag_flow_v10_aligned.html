RAG ne zaman seçilmeli?
✓ Şirket içi doküman veya güncel veriye erişim gerektiğinde
✓ Modeli yeniden eğitmek pahalı veya mümkün değilse
✓ Bilgi tabanı sık güncelleniyorsa
✓ Kaynaklı, izlenebilir yanıt vermek öncelikliyse
✓ Hallucination riskini minimize etmek gerekiyorsa
✓ Bağlam penceresi yeterli değilse
Recall & Precision — Belge Erişiminde Denge
RAG sisteminde sorgulama kalitesini belirleyen iki temel metrik
Recall (Geri Çağırma)
İlgili belgelerin ne kadarı getirildi?
RET_REL / REL
Düşük Recall → Bazı ilgili belgeler eksik, yanıt eksik kalır.
Precision (Hassasiyet)
Getirilen belgelerin ne kadarı gerçekten ilgiliydi?
RET_REL / RET
Düşük Precision → Alakasız belgeler LLM bağlamını kirletir.
İlgili Belgeler (REL)
Getirilen Belgeler (RET)
Kesişim (RET_REL)
Düşük Recall → Eksik Yanıt
İlgili belgelerin bir kısmı getirilemiyor. LLM eksik bağlamla yanıt üretiyor, yanıt yetersiz kalıyor.
Düşük Precision → Kirli Bağlam
Alakasız belgeler bağlama karışıyor. LLM gürültülü veriden yanıt üretiyor, hallucination riski artar.
Chunk boyutu küçüldükçe
Precision artar (daha odaklı), Recall düşer (daha az bağlam). İnce ayar gerektiren bir denge.
RAG sisteminde hedef
İlgili belgelerin büyük bölümünü getirirken (yüksek Recall) alakasız belgeleri dışarıda tutmak (yüksek Precision).
