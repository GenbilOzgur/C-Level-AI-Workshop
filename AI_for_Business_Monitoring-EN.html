PRODUCT 04 Meeting Intelligence & RAG Assistant

Meeting Intelligence Assistant

A meeting assistant that takes Teams meeting transcripts, strips out confidential content, runs them through Azure AI Content Safety steps, and chunks and stores them with Azure AI Search. The Assistant on Foundry runs RAG search via index metadata by meeting date and participants — so each user only searches within the meetings they attended. When the Teams meeting ends, the process is triggered automatically via the Graph API.

Automatic
Processing Pipeline

Teams Transcripts

Transcripts of Teams meetings are taken as the source and fed into the processing pipeline.

Privacy Stripping & Content Safety

Confidential content is removed; the text is made safe by passing through Azure AI Content Safety steps.

Azure AI Search — Chunk & Index

Content is split into chunks and stored in Azure AI Search; indexed with date and participant metadata.

Foundry Assistant — RAG

The Assistant on Foundry grounds and runs RAG search using date and participant indexes.

Participant-Based Access

Each user searches only within meetings they attended; access is limited to the participant list.

Automation via Graph API

The moment a meeting ends, the process is triggered automatically via Microsoft Graph API; no manual step needed.

Supported Infrastructure
Microsoft Teams + Azure AI Content Safety + Azure AI Search + Foundry + Graph API

Read-heavy, action-light: an observability layer that watches agent state without orchestrating agent logic.

Real-time
Monitoring Capabilities

Agent Detail Panel

LLM models, MCP servers, function-calling tools and RAG knowledge bases for each agent, in one slide-over panel.

Live Log Stream

Server-sent execution logs with type filters (model, function, MCP, error) and expandable JSON payloads, newest first.

Inline Actions
